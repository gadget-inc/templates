# Shop Mini Globe Demo

A Package Tracker Shop Mini that visualizes shipment routes on an interactive 3D globe. This template demonstrates how to wire a Shop Mini to a Gadget backend with authenticated routes, third-party APIs, and per-buyer data persistence. This template includes:

1. Shop Mini frontend: Pre-built React + Three.js mini in `minis/shop-mini-globe-demo/` that animates arcs between tracking events.
2. Gadget HTTP routes: `POST /shop-mini/auth`, `POST /shop-mini/tracking`, and `GET /shop-mini/shipments`.
3. Data models: `miniBuyer` (1:1 with a session) and `shipment` (1:many with a buyer).
4. Custom access role: `shop-mini-buyers` role with Gelly filters that scope reads to the current buyer's records.

## Template setup

Follow these steps to get the template working:

1. [Connect your Gadget app to Shopify](https://docs.gadget.dev/guides/plugins/shopify/quickstarts/shopify-quickstart) using the built-in Shopify plugin.

2. Pull your Gadget project locally using `ggt dev` ([Gadget CLI](https://docs.gadget.dev/reference/ggt#ggt-reference)).

3. Install dependencies for the mini:

   ```bash
   cd minis/shop-mini-globe-demo
   npm install
   ```

4. Generate your Shop Minis API key:

   ```bash
   npx shop-minis setup --regenerate-key
   ```

   Sign into Shopify and select your org. This writes a `.env` containing your `SHOP_MINIS_ADMIN_API_KEY`.

5. Configure environment variables in your Gadget app (via the Gadget dashboard or `ggt var set <NAME> <VALUE>`):

   | Variable | Purpose |
   | --- | --- |
   | `SHOP_MINIS_ADMIN_API_KEY` | Copied from `minis/shop-mini-globe-demo/.env` after running `shop-minis setup`. Used to verify Shop Mini tokens. |
   | `SHIP24_API_KEY` | [Ship24](https://www.ship24.com/) API key. Used by `POST /shop-mini/tracking` to look up shipment events. |
   | `GEOCODING_API_KEY` | [geocode.maps.co](https://geocode.maps.co/) API key. Used to convert event location strings into latitude/longitude. |
   | `GADGET_ENVIRONMENT_JWT_SIGNING_KEY` | Auto-generated by Gadget. Used to sign the JWT returned by the auth route. |

6. Update the Gadget domain in the mini. Search-and-replace `shop-mini-globe-demo--development.gadget.app` with `<your-app-name>--development.gadget.app` in:
   - `minis/shop-mini-globe-demo/src/App.tsx` (`TRACKING_API_BASE`)
   - `minis/shop-mini-globe-demo/src/hooks/useAuth.ts` (`AUTH_API`)
   - `minis/shop-mini-globe-demo/src/manifest.json` (under `trusted_domains`)

   Leave `cdn.jsdelivr.net` in `trusted_domains`. It is required by the globe component for loading texture assets.

7. Run the mini from `minis/shop-mini-globe-demo/` with `npx shop-minis dev`. Press `i` for the iOS Simulator, `a` for the Android Emulator, or `q` for a QR code to test on a physical device.

## App workflow summary

1. Buyer opens the Shop Mini

   The `useAuth` hook calls `useGenerateUserToken()` from the Shop Minis SDK to get a short-lived Shop Mini token, then POSTs that token to `POST /shop-mini/auth`. The route verifies the token with Shopify's Admin API, upserts a `miniBuyer` record using the verified `publicId`, creates or reuses a session with the `shop-mini-buyers` role, and returns a Gadget-signed JWT.

2. Buyer enters a tracking number

   The mini sends the number to `POST /shop-mini/tracking` with the JWT. The route fetches events from Ship24, sorts and de-duplicates them, then geocodes each location via geocode.maps.co (paced at 1 request per ~1.1s to stay within the free tier). The resolved path is upserted onto a `shipment` record scoped to the buyer.

3. Globe renders the shipment

   The mini draws an arc per leg, focusing the camera on each segment as it animates. Type `TEST` as the tracking number to skip Ship24 and geocoding and draw a randomized demo path.

4. Buyer views their full history

   Tapping View Full History calls `GET /shop-mini/shipments`, which returns the buyer's 100 most-recent shipments. The mini draws every leg simultaneously with a brief stagger and fades them out.

## Key features

- Models

  - `miniBuyer`: One record per Shop App user. Fields: `publicId` (unique identifier from Shopify), `state` (`VERIFIED` or `GUEST`), `tokenExpiresAt`, `session` (hasOne), `shipments` (hasMany).
  - `shipment`: One record per tracked package per buyer. Fields: `trackingNumber`, `path` (JSON array of `{ location, lat, lng }` points), `lastFetchedAt`, `miniBuyer` (belongsTo).
  - `session`: Extended with a `miniBuyer` belongsTo relationship so the auth flow can hydrate sessions.

- HTTP Routes

  - `POST /shop-mini/auth`: Exchanges a Shop Mini token for a Gadget-signed JWT. Sets `aud` to your app's primary domain and `sub` to the session ID so Gadget auto-populates `session` on subsequent API calls.
  - `POST /shop-mini/tracking`: Looks up a tracking number on Ship24, geocodes the events, and persists the shipment.
  - `GET /shop-mini/shipments`: Returns the current buyer's 100 most-recent shipments.

- Frontend

  - `web/routes/_app._index.tsx`: Embedded Shopify admin dashboard with an `AutoTable` of installed shops.
  - `minis/shop-mini-globe-demo/src/App.tsx`: Shop Mini entrypoint that authenticates the buyer and orchestrates the tracking flow.
  - `minis/shop-mini-globe-demo/src/components/Globe.tsx`: Three.js globe component that renders shipment routes as animated arcs.
  - `minis/shop-mini-globe-demo/src/hooks/useAuth.ts`: React hook that exchanges the Shop Mini token for a Gadget JWT and refreshes it when expired.

- Access control

  - `shop-mini-buyers`: Custom role assigned to sessions during authentication. Grants read on `miniBuyer` and `shipment` via Gelly filters that scope access to the current session's `miniBuyer`.

## How to test it

1. Start the mini

   From `minis/shop-mini-globe-demo/`, run `npx shop-minis dev` and open in the iOS Simulator (`i`), Android Emulator (`a`), or via QR code (`q`).

2. Try a demo path

   Type `TEST` as the tracking number to draw a randomized demo path without burning Ship24 or geocoding API quota.

3. Track a real shipment

   Enter a real tracking number to fetch live events from Ship24 and watch the geocoded path animate on the globe.

4. View full history

   Tap View Full History to render every leg of the buyer's 100 most-recent shipments.
